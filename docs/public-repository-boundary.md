# Public Repository Boundary

This repository contains only public-facing material.

## Allowed

- Reviewed founder-facing documentation
- Approved examples
- Sample templates
- Open-source tools
- Public product outputs

## Excluded

- Raw research
- Private source notes
- Interview transcripts without explicit permission
- Source scoring
- Internal knowledge graph data
- Automation internals
- AI prompt libraries
- Internal operating notes
- Unverified legal, tax, visa, or banking claims

## Promotion Rule

Material may move from the private repository to the public repository only when it has been reviewed, cleaned of private material, and written in founder-safe language.

## Public-Safe Examples

Public examples may show:

- the shape of a founder-facing answer
- a sanitized service map
- a reusable checklist structure
- a template for future research

Public examples must not reveal:

- private source scoring
- internal source IDs
- raw evidence files
- private notes
- unpublished interviews
- internal prompt or automation design

## Language Rule

Public outputs should distinguish between:

- a source map
- a verified requirement
- a practical checklist
- a final recommendation

Do not present a source map as final advice.
