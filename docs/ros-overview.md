# Research Operating System Overview

The Founder Intelligence Platform uses a Research Operating System (ROS) to convert raw information into reusable founder intelligence.

## Public Summary

ROS is a disciplined workflow for research:

1. Capture
2. Normalize
3. Verify
4. Score
5. Summarize
6. Link
7. Publish
8. Review

The public repository does not contain the private ROS database or raw evidence. It contains only selected public outputs and examples.

## Core Principle

Research once. Publish everywhere.

The same verified research should be reusable across:

- handbook entries
- checklists
- founder answers
- templates
- public examples
- future tools

## Public/Private Split

Private system:

- raw evidence
- private source notes
- review notes
- knowledge graph
- internal workflows
- source scoring
- automation internals

Public surface:

- reviewed documentation
- public examples
- public templates
- selected founder-facing outputs
- open-source tools

## Why This Matters

Founder research often becomes fragile because it lives in browser tabs, chats, downloads, and memory.

ROS keeps research structured so public outputs can be:

- traceable
- reusable
- reviewable
- easier to update
- safer to publish

## Boundary

This overview is public. It describes the method, not the private operating database.
